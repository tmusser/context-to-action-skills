# Examples

These examples show how messy workplace context becomes facts, asks, decisions, updates, actions, or replies while preserving uncertainty.

| Example | Primary skill | Workflow | What it proves |
| --- | --- | --- | --- |
| [reduce-to-facts-vendor-risk.md](reduce-to-facts-vendor-risk.md) | `reduce-to-facts` | Dense source → fact ledger | Separates facts from unsupported claims before a launch decision. |
| [reduce-to-facts-source-expanded.md](reduce-to-facts-source-expanded.md) | `reduce-to-facts` | Dense source + outside evidence → claim status | Distinguishes exact support, support in substance, and overstatement. |
| [messy-thread-to-follow-up.md](messy-thread-to-follow-up.md) | `follow-up-draft` | Slack thread → clear ask → follow-up | Turns a messy thread into a safe follow-up without overstating certainty. |
| [gmail-buried-obligation.md](gmail-buried-obligation.md) | `clear-ask` | Email chain → buried obligation → reply | Surfaces a hidden deadline and owner gap from polite email. |
| [leadership-status-update.md](leadership-status-update.md) | `status-update` | Workweek context → leadership update | Converts leadership context into a short update with risk and audience sensitivity. |
| [post-meeting-follow-up.md](post-meeting-follow-up.md) | `meeting-to-actions` | Meeting transcript → action record → follow-up | Extracts owners, actions, blockers, and open loops from meeting notes. |
| [morning-brief.md](morning-brief.md) | `brief-me` | Overnight context → brief | Gives a quick catch-up that highlights what changed and what needs attention. |
| [decision-with-data.md](decision-with-data.md) | `decision-brief` | Data + notes → decision snapshot | Frames a decision with data without overstating certainty or readiness. |
| [async-slack-clear-ask.md](async-slack-clear-ask.md) | `clear-ask` | Slack thread → async meeting → ask | Treats a Slack thread like an async meeting and names the real decision gap. |

## Strong Examples Should Show

- raw context
- skill invocation
- output
- what changed
- why this beats a generic summary

## How To Use These

- Pick the example closest to your messy context.
- Paste similar source material into the assistant.
- Ask for the named skill.
- Start with `reduce-to-facts` when the source is long, ambiguous, contradictory, or high-stakes.

## Suggested Reading Order

- Start with [reduce-to-facts-vendor-risk.md](reduce-to-facts-vendor-risk.md).
